# Reports — Mandatory at Milestone 5

Every milestone-5 structure pass writes a trade-spec HTML report. Every closed trade writes a P&L report. Cards for social are generated by the Next.js share endpoint.

---

## Report Inventory

| Report | Template | Output |
|---|---|---|
| Trade Spec | `.pi/skills/html-report/trade-specification-template.html` | `reports/{ticker}-evaluation-{YYYY-MM-DD}.html` |
| P&L | `.pi/skills/html-report/pnl-template.html` | `reports/pnl-{TICKER}-{YYYY-MM-DD}.html` |
| Share PnL Card | `next/og` (Satori), 1200×630 PNG | `web/app/api/share/pnl/route.tsx` |

Reference rendered output: `reports/goog-evaluation-2026-03-04.html`.

---

## Trade Spec — Required Sections

1. Header + four-gates summary
2. Summary (decision, conviction, size)
3. Milestone pass/fail table
4. Dark Pool analysis (actual + interpolated)
5. Options Flow analysis (P/C, side bias, OI changes)
6. Context (analyst ratings, seasonality, news)
7. Structure & Kelly math
8. Spec (legs, fills, net debit/credit)
9. Thesis & Risk
10. Four Gates re-statement

---

## Key Formula

```
Return on Risk = P&L / Capital at Risk
```

Capital at Risk = max-loss for defined-risk structures, full debit for long premium, strike × 100 + net premium for short puts / risk reversals.

---

## Output Discipline

- Always state probabilities — no certainty claims.
- Flag uncertainty explicitly (e.g., LOW confidence on dark pool data before 11 AM).
- Failing gate = stop, name the gate.
- **Never rationalize a bad trade.** If the report doesn't pass the four gates, the decision is NO_TRADE.
